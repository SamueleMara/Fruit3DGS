1. DBSCAN clustering

sklearn.cluster.DBSCAN does not natively use the GPU.

Options if you want GPU acceleration:

cuML
 from NVIDIA’s RAPIDS suite:

from cuml.cluster import DBSCAN as cuDBSCAN
clusters = cuDBSCAN(eps=0.03, min_samples=1).fit(coords)
labels = clusters.labels_


Very similar API to sklearn.

Works on NVIDIA GPUs.

HDBSCAN on GPU (via cuML or hdbscan with RAPIDS).

2. 2D projection and mask operations

Right now you do everything with NumPy and PIL, which are CPU-only.

You could use PyTorch or CuPy to move computations to GPU:

Convert masks and coordinates to GPU tensors.

Perform projection checks and masking using PyTorch tensor operations.

Example (simplified):

import torch

masks_gpu = torch.tensor(mask, device='cuda')
px_gpu = torch.tensor([px], device='cuda')
py_gpu = torch.tensor([py], device='cuda')
inside_mask = masks_gpu[py_gpu, px_gpu] != 0


This can accelerate masking operations for large numbers of points and images.

3. Optimization

scikit-optimize (skopt) runs on CPU only.

If you wanted GPU, you’d need a GPU-compatible Bayesian optimization library like BoTorch.

✅ Summary

GPU-friendly parts: clustering (cuML DBSCAN) and mask computations (PyTorch/CuPy).

CPU-only parts: skopt Bayesian optimization, PIL image I/O, Python loops.
